## Issues Found

### Still Open from P3a

1. **F014 - Tilde Path Security Bypass (HIGH)**: `~` paths still bypass workspace validation. `~/.docker/config.json` was read without blocking.
2. **A014 - System Prompt Setter Bug (MEDIUM)**: `--system-prompt` flag causes "Error: can't set attribute". The property setter on the config object is broken.

### New Issues Found in P3b

3. **N028/N029 - Finance Mode Tool Execution (HIGH)**: In `fin` mode, LLM-generated `<tool_call>` XML is printed as raw text instead of being executed. The tool dispatch pipeline in finance mode appears broken. Same issue does NOT affect coding mode.

4. **M014 - Empty Response Bug (MEDIUM)**: "Generated content:" label shown but body is empty. This is the same empty response rendering bug noted in P3a O011. Affects text-only responses where the LLM returns content that doesn't render properly.

5. **P010 - Config Accepts Unknown Keys (LOW)**: `/config set nonexistent_key value` succeeds without validation. Unknown configuration keys should be rejected or at least warned about.

6. **O017 - Think Toggle Semantics (LOW)**: `/think on` toggles (if already ON, switches to OFF). Expected behavior: `/think on` should always enable, `/think off` should always disable. Current behavior treats the argument as a toggle trigger regardless of the argument value.

### Fixed Since P3a

- J005: Duplicate team creation now properly blocked
- J014: "Always allow" now persists across turns
- G022, G025, G026: `/verbose`, `/hooks`, `/arch` commands now implemented
- A013, A015: Headless mode now executes tool calls (no more raw XML)
