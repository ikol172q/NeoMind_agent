## Analysis of Failures

### Real Bugs (confirmed issues in code)

**BUG-1: PARSE FAILED - tool_call parser chokes on `<thinking>` inside `<tool_call>` tags**
- Affected: F002, F003, F004, F005
- The DeepSeek model emits `<tool_call><thinking>...</thinking>{JSON}</tool_call>` and the parser fails
- Visible to user: `[agentic] tool_call tag present but PARSE FAILED`
- Severity: HIGH - causes tool execution failures, raw error output shown to user

**BUG-2: Chinese echo command generates malformed bash - `echo你好世界` (no space)**
- Affected: C002, C003, C004 (cascading)
- The LLM generates `echo你好世界` without space, bash interprets as unknown command, exit code 127
- LLM retries repeatedly, creating cascading failures across subsequent scenarios
- Severity: MEDIUM - LLM prompt issue, not code bug

**BUG-3: Spinner characters visible in final rendered output**
- Affected: C003-C007, F001, F007, F008, G001, G011, G013, G014
- Spinner animation chars (e.g., `⠸`, `⠧`, `⠋`) remain visible after response completes
- Note: Some of these are test timing artifacts (capture during processing), but several are confirmed visible in final output
- Severity: LOW-MEDIUM - cosmetic but looks unpolished

**BUG-4: G001 /help was consumed by pending permission prompt**
- The test script sent `/help` while a permission prompt from a previous interaction was still active
- `/help` was interpreted as a permission response ("Denied"), not as a slash command
- This is partly a test sequencing issue, but also shows that pending permission prompts can trap user input
- Severity: LOW - edge case

### False Positives (test artifacts, not real bugs)

- F006 (rm -rf /) PASS is correct - the system blocked the command silently
- F008, G014 spinner chars are from tmux capturing mid-transition during /exit
- G005-G009 showed "no server running" because tmux session was being torn down/recreated; the commands themselves worked
- G012-G014 "Content repeated 3x" for `main.py (326 lines)` is from the LLM reading the file multiple times (normal behavior), not a display bug
