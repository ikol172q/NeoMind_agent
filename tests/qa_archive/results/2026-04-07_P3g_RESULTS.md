### Top Issues Found

1. **Permission prompt input collision** (systemic): When the agent chains tool calls and shows permission prompts, user inputs typed during the prompt are treated as y/n/a responses. Non-matching text causes auto-denial. This is the #1 cause of WARN results across all batches.

2. **Write tool does not write files** (known bug): Agent generates "Generated content:" with beautiful code but does NOT write to disk. Confirmed again in T29-T30.

3. **Compact memory loss** (by design?): /compact reduces token count but loses specific details like user names. Workplace info partially preserved. Compact summary quality varies.

4. **Search routing** (design issue): "Search for X in this project" triggers web search (DuckDuckGo) instead of local file grep. The grep intent detection pattern is too narrow.

5. **Tool XML parse failures**: LS tool calls sometimes produce `<tool_call>` XML that the parser cannot handle. Known parser limitation.

6. **`find -maxdepth2`** (command mangling): Space before numeric argument consistently dropped, causing `find: -maxdepth2: unknown primary or operator`.

### Strengths Confirmed

- **Security classification**: Dangerous commands correctly classified HIGH risk, safe commands LOW risk
- **Slash commands**: /skills, /context, /stats, /cost, /flags, /doctor, /checkpoint, /rewind, /compact, /help all work reliably
- **Mode switching**: Stable across coding/chat/fin modes
- **Error recovery**: Graceful handling of incomplete commands (Write, Edit, Run:, Search with no args)
- **40-turn stability**: No crashes, no hangs, no memory exhaustion across the longest test
- **/deep and /compare**: Produce high-quality analytical content
- **Checkpoint/rewind**: Reliable even at scale (30+ turns)
